# 🎓 Quiz & Document Management System (LMS)

A robust, PHP-based Learning Management System designed to bridge the gap between Administrators and Students. This application allows for dynamic content creation, secure document handling, and an interactive examination system.

## 🚀 Project Overview

This project serves as a dual-purpose platform for educational and administrative tasks. It features a split-role architecture:
* **Administrator:** Controls user data, manages quizzes, oversees document uploads, and handles dynamic site pages (CMS).
* **User (Student):** Attempts time-bound quizzes, views personal results history, and manages personal document uploads.

## 🛠️ Key Functionalities

### 1. Secure Authentication
* **Encryption:** Passwords are secured using `password_hash` with BCRYPT.
* **Role-Based Redirects:** The system automatically identifies the user role upon login and redirects them to the appropriate dashboard.

### 2. Administrator Suite
* **User Management:** Complete CRUD (Create, Read, Update, Delete) operations for student accounts.
* **Quiz Factory:** Ability to create quizzes and add unlimited Multiple Choice Questions (MCQs).
* **Document Monitoring:** A centralized view to manage and verify files uploaded by students.
* **Content Management (CMS):** A dynamic system to create "About Us," "FAQ," or "Policy" pages without touching the code.

### 3. Advanced Quiz Engine
* **Live Countdown Timer:** Each quiz features a 10-minute timer. The system triggers an **auto-submit** function when the time reaches zero.
* **Anti-Cheat Shuffle:** Questions are displayed in a randomized order (`ORDER BY RAND()`) for every attempt.
* **Instant Grading:** Real-time score calculation and pass/fail status immediately following submission.
* **Performance Tracking:** Students can access a "Quiz Results" tab to view their historical performance data.

### 4. File Management
* **Validation:** Restricts file types to PDF, JPG, and PNG with a 2MB size limit.
* **Storage Logic:** Files are renamed using a combination of timestamps and unique IDs to prevent naming conflicts on the server.

### 5. Communication
* **SMTP Integration:** The contact form utilizes **PHPMailer** to send inquiries directly to the administrator's email.

## 📂 Core File Structure

* `config.php`: Central database configuration.
* `admin.php`: Main entry point for administrators.
* `user_quiz.php`: Dashboard for students to select available tests.
* `start_quiz.php`: The active testing interface with the JavaScript timer.
* `submit_quiz.php`: The backend engine that grades tests and saves results.
* `seeder.php`: Essential tool for initializing database tables and dummy data.

---

## ⚙️ Setup Instructions

1.  Move the project files to your local server directory (e.g., `C:\xampp\htdocs\admin`).
2.  Ensure you have a MySQL database named `test` created.
3.  Execute `config.php` to verify the connection.
4.  Run `seeder.php` via your browser (`localhost/admin/seeder.php`) to build the tables and insert the users listed above.
5.  Login using the Admin or Student credentials to begin testing.

---

**Security Note:** Always use the `logout.php` link to properly terminate sessions. For production, ensure the `seeder.php` file is removed.
